# 录制帧率优化计划

## 问题描述

当前录制的视频成品帧率较低（约20帧），需要优化录制参数以提高帧率。

## 根本原因分析

### 当前实现问题

1. **Canvas捕获帧率**：`recordCanvas.captureStream(60)` - 设置为60fps ✓
2. **MediaRecorder时间片**：`mediaRecorder.start(1000)` - 设置为1000ms（1秒）✗

**问题核心**：

* `MediaRecorder.start(1000)` 中的 `1000` 参数表示每1000毫秒（1秒）才收集一次数据块

* 即使canvas以60fps捕获画面，MediaRecorder也只会在每秒结束时才写入一个数据块

* 这导致最终视频帧率大幅降低到约20fps

## 优化方案

### 方案一：减小MediaRecorder时间片（推荐）

**修改位置**：`src/App.jsx` 第668行

**修改前**：

```javascript
mediaRecorder.start(1000);
```

**修改后**：

```javascript
mediaRecorder.start(100); // 改为100ms
```

**预期效果**：

* MediaRecorder每100ms收集一次数据块

* 帧率将从约20fps提升到接近60fps

* 文件大小可能略微增加（但可以接受）

**优点**：

* 修改简单，只需改动一行代码

* 性能影响小

* 帧率提升明显

**缺点**：

* 文件大小可能略微增加（约10-20%）

***

### 方案二：动态调整时间片（备选）

如果方案一效果不理想，可以进一步优化：

**实现思路**：

1. 根据录制时长动态调整时间片
2. 短时间录制使用较小时间片（100ms）
3. 长时间录制使用稍大时间片（200-250ms）以控制文件大小

**实现位置**：`src/App.jsx` 第668行附近

***

## 实施步骤

### 步骤1：修改MediaRecorder时间片

* 文件：`src/App.jsx`

* 行号：第668行

* 操作：将 `mediaRecorder.start(1000)` 改为 `mediaRecorder.start(100)`

### 步骤2：测试验证

* 启动开发服务器：`npm run dev`

* 进行录制测试（录制5-10秒）

* 检查输出视频的帧率是否提升到接近60fps

* 验证视频质量是否保持良好

### 步骤3：性能检查（可选）

* 监控录制过程中的CPU和内存使用情况

* 如果性能下降明显，可以尝试将时间片调整为150ms或200ms

## 预期结果

* **帧率**：从约20fps提升到50-60fps

* **视频流畅度**：明显提升，动作更流畅

* **文件大小**：可能增加10-20%，但质量提升值得

* **性能影响**：轻微，可接受

## 可能出现的问题及测试用例

### 问题1：文件大小增加过多

**测试用例**：

* 录制1分钟视频

* 检查文件大小是否超过合理范围（例如超过500MB）

**解决方案**：

* 将时间片调整为150ms或200ms

* 或考虑降低canvas捕获帧率到30fps

### 问题2：性能下降明显

**测试用例**：

* 在配置较低的设备上测试

* 监控CPU和内存使用情况

* 检查浏览器是否出现卡顿

**解决方案**：

* 降低canvas捕获帧率到30fps

* 增加MediaRecorder时间片到200-250ms

### 问题3：视频编码兼容性问题

**测试用例**：

* 在不同浏览器中测试（Chrome、Firefox、Edge）

* 检查视频是否能正常播放

**解决方案**：

* 如果某些浏览器不支持vp9编码，可以添加编码格式检测

* 回退到vp8编码或其他兼容格式

## 风险评估

* **风险等级**：低

* **影响范围**：仅影响录制功能

* **回滚难度**：简单，只需改回原来的1000ms即可

## 后续优化建议

1. **添加录制质量设置**：允许用户选择录制质量（高/中/低）
2. **实时帧率显示**：在录制时显示当前帧率
3. **智能时间片调整**：根据设备性能自动调整时间片
4. **编码格式优化**：尝试使用更高效的编码格式（如H.264）
