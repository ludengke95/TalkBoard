# 修复新建演讲页滚动问题

## 问题分析

在 `handleAddSlide` 函数中：

1. `newSlideIndex = slides.length`（新演讲页的索引）
2. 调用 `handleSelectSlide(newSlideIndex)` → `scrollToPage(newSlideIndex)`
3. `scrollToPage` 函数检查 `pageIndex >= slides.length`
4. 但此时 `slides` 状态还未更新（`setSlides` 是异步的），`slides.length` 还是旧值
5. 导致 `pageIndex >= slides.length` 为 `true`，函数提前返回，无法滚动

## 解决方案

使用 `useEffect` 监听 `slides` 数组的变化，当检测到新增演讲页时，自动滚动到最后一个。

## 实现步骤

### 步骤 1：添加 useRef 记录上一次 slides 长度

**文件**: `src/App.jsx`

```javascript
// 记录上一次 slides 长度，用于检测新增
const prevSlidesLengthRef = useRef(slides.length);
```

### 步骤 2：添加 useEffect 监听 slides 变化

**文件**: `src/App.jsx`

```javascript
// 监听 slides 变化，新增演讲页时自动滚动到最后一个
useEffect(() => {
  // 检测是否新增了演讲页
  if (slides.length > prevSlidesLengthRef.current && slides.length > 0) {
    // 滚动到最后一个演讲页
    handleSelectSlide(slides.length - 1);
  }
  // 更新记录的长度
  prevSlidesLengthRef.current = slides.length;
}, [slides, handleSelectSlide]);
```

### 步骤 3：移除 handleAddSlide 中的滚动逻辑

**文件**: `src/App.jsx`

将：

```javascript
// 更新演讲页状态
const newSlideIndex = slides.length;
setSlides((prev) => [...prev, newSlide]);
handleSelectSlide(newSlideIndex);
```

改为：

```javascript
// 更新演讲页状态
setSlides((prev) => [...prev, newSlide]);
```

### 步骤 4：更新依赖数组

从 `handleAddSlide` 的依赖数组中移除 `handleSelectSlide`：

```javascript
}, [slides, calculateSlideSize, createSlideElement]);
```

## 测试要点

1. 新建第一个演讲页，验证是否能正常滚动到该页
2. 新建多个演讲页，验证每次都能滚动到新建的演讲页
3. 验证翻页、删除演讲页等功能是否正常（删除不应触发滚动）
