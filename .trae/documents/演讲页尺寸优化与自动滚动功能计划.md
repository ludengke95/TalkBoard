# 演讲页尺寸优化与自动滚动功能计划

## 需求分析

### 1. 演讲页固定尺寸

当前实现是根据窗口宽度动态计算演讲页尺寸，需要改为固定尺寸：

| 比例   | 宽度   | 高度   |
| ---- | ---- | ---- |
| 16:9 | 1920 | 1080 |
| 4:3  | 1280 | 960  |
| 3:4  | 960  | 1280 |
| 9:16 | 1080 | 1920 |
| 1:1  | 800  | 800  |

### 2. 新增演讲页自动滚动

新增演讲页后，页面应自动滚动到新增的演讲页位置。

***

## 实现步骤

### 步骤 1：修改 `calculateSlideSize` 函数

**文件**: `src/App.jsx`

将动态计算改为固定尺寸映射：

```javascript
const calculateSlideSize = useCallback(() => {
  // 固定尺寸映射表
  const sizeMap = {
    '16:9': { width: 1920, height: 1080 },
    '4:3': { width: 1280, height: 960 },
    '3:4': { width: 960, height: 1280 },
    '9:16': { width: 1080, height: 1920 },
    '1:1': { width: 800, height: 800 },
  };
  
  // 根据当前画面比例返回对应尺寸，默认 16:9
  return sizeMap[aspectRatio] || sizeMap['16:9'];
}, [aspectRatio]);
```

### 步骤 2：修改 `handleAddSlide` 函数

**文件**: `src/App.jsx`

在添加演讲页后调用 `scrollToPage` 滚动到新增的演讲页：

```javascript
const handleAddSlide = useCallback(() => {
  // ... 现有创建逻辑 ...
  
  setSlides((prev) => [...prev, newSlide]);
  
  // 新增：自动滚动到新增的演讲页
  // 需要使用 setTimeout 确保状态更新后再滚动
  setTimeout(() => {
    scrollToPage(slides.length);
  }, 100);
}, [slides, calculateSlideSize, createSlideElement, scrollToPage]);
```

## 测试要点

1. **尺寸测试**：切换不同画面比例后，新增演讲页验证尺寸是否正确
2. **自动滚动测试**：新增演讲页后，验证页面是否自动滚动到新增的演讲页
3. **兼容性测试**：验证现有演讲页列表、翻页功能是否正常工作
